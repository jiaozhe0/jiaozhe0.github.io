从对象说起 #

在《面向对象分析与设计》这本书中，Grady Booch 替我们做了总结，他认为，从人类的认知角度来说，对象应该是下列事物之一:

  • 一个可以触摸或者可以看见的东西
  • 人的智力可以理解的东西
  • 可以指导思考或者行动（进行想象或施加动作）的东西

编程语言角度的对象特征：

  • 对象具有唯一标识性：即使完全相同的两个对象，也并非同一个对象。
    • 各种语言对象唯一标识性都是用内存地址体现的，对象具有唯一标识的内存地址
  • 对象有状态(属性)：对象具有状态，同一对象可能处于不同状态之下。
  • 对象具有行为(方法)：即对象的状态，可能因为它的行为产生变迁。
  • 它是一种结构化的数据类型：
    • 数据类型：占据内存区域大小和存储在该内存区域的数据类型
    • 如 C 语言的 short 类型：占据 2 个字节的内存区域，并存储整数
    • 结构体：将若干个数据项汇聚到一处并赋予其名字后所形成的一个整体 一旦定义完结构体，就可以把结构体当作是一个数据结构

Javascript 里的对象 #

  • ECMA-262 将对象定义为一组属性的无序的集合
  • 属性并非简单名/值对，js 使用一些内部特性来描述属性的特征
  • 独有的特性：对象具有高度的动态性，js 赋予了使用者在运行时为对象添改状态和行为的能力

创建对象 #

  • new Object()
  • 对象字面量
  • Object.create()
  • Object.defineProperty()
// 构造函数
var obj = new Object();
// 字面量
const obj1 = {
  name: ''
}

// Object.create()
const obj3 = Object.create(null) // 创建一个没有原型的对象

// Object.defineProperty()
const obj4 = Object.defineProperty({}, 'name', {
  value: '对象',
  writable: true,
  enumerable: true,
})

对象创建的设计模式 #

  • 工厂模式
    • 缺乏类型标识
  • 构造函数模式
  • 原型模式
javascript
// 工厂模式
function createPerson(name, age) {
  const person = new Object();
  person.name = name;
  person.age = age;
  person.getName = function () {
    return person.name;
  };
  return person;
}
const xm = createPerson("xm", 12);

// 构造函数
function Person(name, age) {
  this.name = name;
  this.age = age;
  this.getName = function () {
    return this.name;
  };
}
const xh = new Person("xh", 13);

// 原型模式

创建私有属性 #

  • 使用 ES13 规范，通过#符号，来定义一个私有属性
  • 使用闭包和 IIFE 创建私有属性
  • 使用 WeakMap 创建私有属性
  • 使用 Symbol 创建私有属性
  • 使用闭包和getter 访问器属性
javascript
// es13
class Person {
  #name;
  constructor() {
    this.#name = "John";
  }
  getName() {
    return this.#name;
  }
}
// --- 用闭包和 IIFE 创建私有属性 ---
// 字面量
const person = (function () {
  let name = "前端";
  return {
    getName: function () {
      return name;
    },
  };
})();
// 构造函数
const Person = function () {
  var name = "私有名字";
  this.getName = function () {
    return name;
  };
};
// 原型模式
const Person = function () {
  var name = "私有名字";
  this.getName = function () {
    return name;
  };
};
Person.prototype = () => {
  let age = 12;
  return {
    getAge() {
      return age;
    },
  };
};

// --- WeakMap ---
let Person
{
  const _weakMap = new WeakMap()
  Person = function() {
    _weakMap.set(this, {name: '私有属性'});
  }
  Person.prototype.getName = funcion() {
    return _weakMap.get(this).name
  }
}
const person = new Person();
console.log(person.name) // undefined
console.log(person.getName()) // 私有属性

// --- symbol---
let Person
{
  const _privateProps = Symbol()
  Person = function() {
    this[_privateProps] = 'symbol'
  }
  Person.prototype.getName = function() {
    return this[_privateProps]
  }
}
var person = new Person()

属性类型 #

上面创建的每个对象都有自己的属性和方法，这些属性在创建时都带有一些特征值，这些特征值仅供内部使用，在 JavaScript 中不能直接访问他们，按规定这些特征值需要放在两对方括号中，例如[[Enumerable]]

数据属性 #

  • [[Value]]:属性实际的值, 默认为:undefined
  • [[Enumerable]]:表示属性是否可以枚举(for-in 循环,Object.keys())
    • 默认为:false
    • 如果直接定义在对象上的属性的这个特性都是 true
  • [[Writable]]: 表示属性的值是否可以被修改
    • 默认为:false
    • 所有直接定义在对象上的属性的这个特性都是 true
  • [[Configurable]]：表示属性是否可以通过 delete 删除并重新定义，是否可以修改它的特性，以及是否可以把它改为访问器属性
    • 默认为:false
    • 所有直接定义在对象上的属性的这个特性都是 true
javascript
const obj = {};
Object.defineProperty(obj, value, {
  value: "对象",
});

访问器属性 #

  • [[get]]:获取函数，在读取属性时调用。默认值为 undefined
  • [[set]]:获取函数，在读取属性时调用。默认值为 undefined
  • [[Enumerable]]:表示属性是否可以通过 for-in 循环返回。默认情况下，所有直接定义在对象上的属性的这个特性都是 true
  • [[Configurable]]：表示属性是否可以通过 delete 删除并重新定义，是否可以修改它的特性，以及是否可以把它改为访问器属性。默认情况下，所有直接定义在对象上的属性的这个特性都是 true

访问器属性使得属性在读和写时执行代码，它允许使用者在写和读属性时，得到完全不同的值，它可以视为一种函数的语法糖。

javascript
const obj = {
  _value: "面向对象",
};
Object.defineProperty(obj, "value", {
  get() {
    return this._value;
  },
  set(newVal) {
    this._value = newVal;
  },
});
console.log(obj.value); // 面向对象
obj.value = "基于对象";
console.log(obj.value); // 基于对象
// 🚀：以上是vue2数据响应式，底层实现的语法

let _value = "面向对象";
const obj1 = {
  get value() {
    return _value;
  },
  set value(newVal) {
    _value = newVal;
  },
};
console.log(obj1.value); // 面向对象
obj.value = "基于对象";
console.log(obj1.value); // 基于对象
// 🚀：以上是vue3中ref接口，底层实现的语法

面向对象编程 OOP(Object Oriented Programming) #

  • 面向对象编程是一种编程范式或编程风格
  • 它以类或对象作为组织代码的基本单位
  • 将封装，抽象，继承，多态四个特性，作为代码设计和实现的基石
